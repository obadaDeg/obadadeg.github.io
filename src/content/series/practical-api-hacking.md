This series documents my journey learning API security, covering attack techniques, testing methodology, and tooling. Posts draw from TCM Security's Practical API Hacking course, PortSwigger Academy, and OWASP's API Security Top 10.

## Planned Topics

- What is API hacking and why APIs are uniquely exposed
- Reconnaissance: discovering endpoints, understanding API contracts
- Authentication flaws: broken tokens, weak JWT validation, missing auth entirely
- Broken Object Level Authorization (BOLA/IDOR)
- Excessive data exposure and mass assignment
- Rate limiting and resource exhaustion
- Testing with Burp Suite and Postman
